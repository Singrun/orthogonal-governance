# 贡献指南

## 提交内容类型
- 文稿修订：逻辑、结构、措辞与引用
- 数学修正：推导、公式与参数定义
- 案例补充：遵循匿名化要求的真实场景
- 工程化配置：CI、模板与工具链改进

## 工作流程
- 建议先开 Issue，描述目的与影响范围
- 提交 PR 前运行本仓库的链接与格式校验（CI 将自动执行）
- PR 模板需填写：变更类型、受影响文件、验证方式

## 写作规范
- 统一中文标点与英文缩写间距
- 每文件仅一个一级标题；小节层级从二级开始
- 链接使用规范化文件名；避免指向非规范路径

## 案例匿名化
- 不包含个人隐私、商密与可识别元数据
- 数据脱敏；必要时使用合成数据重现结构问题
- 在 cases 目录按模板提交

- 代码与脚本（如有）：建议采用 MIT/Apache-2.0（后续若引入）
