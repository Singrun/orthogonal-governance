# 案例提交指南（Case Study）

## 提交方式
- 在仓库创建 Issue，标题以“[Case] 机构/系统类型 + 关键症状”命名
- 选择标签：`Case Study`
- 强烈建议匿名化（去除公司/个人可识别信息）

## 建议结构
```
## Context（背景）
系统规模、行业类型、组织结构简述

## Symptoms（症状）
可观测异常（例如：度量漂移、反馈延迟、跨域依赖激增）

## Metrics（度量）
相关指标与趋势（允许用相对变化或区间描述）

## Interventions（干预）
尝试的治理措施（成功/失败均可）

## Outcome（结果）
阶段性结论与当前状态

## Lessons（经验）
与正交治理原则的关系（独立观察轴、去耦合、反熵增等）

## Tags（标签）
如：CSI、OGI、T2阻塞、相变、合谋均衡、审计共线性
```

## 数据与隐私
- 不上传任何受限或敏感数据
- 用抽象描述替代具体名称（如“大型电商平台”、“互联网金融中台”）
